## 2. What Are We Building?

### Core Primitive: Yield Stripping (PT + YT)

The most important derivative is the yield splitter. It takes a user's vault share position and splits it into two tokens:

- **Principal Token (PT)** — redeemable for exactly `1 NUSD` at maturity. Fixed claim. Zero yield risk.
- **Yield Token (YT)** — receives all yield generated by the underlying vault position until maturity. Pure yield play. Can go up or down.

This is how Pendle Finance works, applied to our ERC-4626 vaults.

**Example:**
- User deposits 1,000 NUSD into the Treasury Vault
- Vault gives them 1,000 vault shares (worth $1,000 at current NAV)
- User deposits those vault shares into YieldSplitter
- Gets 1,000 PT (redeemable for 1,000 NUSD at Dec 31 2026) + 1,000 YT
- User sells the PT at a discount (e.g. 970 NUSD) to lock in yield now
- Buyer gets a guaranteed 1,000 NUSD at maturity → implied fixed yield

### Structured Product (Tranches)

A pool of vault collateral split into:
- **Senior tranche** — first claim on collateral, lower yield, capital protection
- **Junior tranche** — residual claim, higher yield, absorbs first losses

For institutions: buy senior for capital preservation. Risk-on: buy junior for leverage on vault yield.

### Credit Vault

Allows borrowing NUSD against vault share collateral. Similar to borrowing DAI against ETH on Maker, but with tokenized treasury vault shares as collateral. Creates yield-bearing collateral positions.

### ETF Wrapper

A single token backed by a basket of multiple yield vaults with fixed allocation weights. Provides diversified exposure across Treasury, Money Market, and future vault types in one token.

## 3. Why These Products?

| Product | Institutional Use Case |
|---------|----------------------|
| PT | Fixed income desk — lock in a yield rate, eliminate rate risk |
| YT | Trading desk — speculate on interest rate moves |
| Senior tranche | Capital-preservation mandates, insurance funds |
| Junior tranche | Yield-seeking investors, family offices |
| CreditVault | Leverage without selling vault position; borrow against reserves |
| ETFWrapper | Index exposure to the full Nexus vault ecosystem |

These are the products that turn a stablecoin + vault into an **institutional fixed income platform**, comparable to on-chain equivalents of T-bill ETFs, CMOs, and repo facilities.

## 5. Contract Designs

### 5.1 YieldSplitter + PrincipalToken + YieldToken

**File locations:**
```
contracts/derivatives/YieldSplitter.sol
contracts/derivatives/PrincipalToken.sol
contracts/derivatives/YieldToken.sol
```

#### PrincipalToken

ERC-20. Each PT represents a claim to exactly `1 underlying asset unit` at the splitter's `maturity` timestamp.

```solidity
contract PrincipalToken is ERC20, AccessControl {
    bytes32 public constant MINTER_ROLE = keccak256("MINTER_ROLE");
    uint256 public immutable maturity;      // UNIX timestamp
    address public immutable underlying;    // NUSD address

    // Only YieldSplitter can mint/burn
    function mint(address to, uint256 amount) external onlyRole(MINTER_ROLE);
    function burn(address from, uint256 amount) external onlyRole(MINTER_ROLE);

    // After maturity: holder burns PT to redeem 1 NUSD per PT
    function redeem(uint256 amount) external;
}
```

Key invariant: `PT.totalSupply() == NUSD locked in splitter for this maturity`.

#### YieldToken

ERC-20. Each YT accrues yield from the vault until maturity. After maturity, YT has no value — all remaining yield was captured.

```solidity
contract YieldToken is ERC20, AccessControl {
    bytes32 public constant MINTER_ROLE = keccak256("MINTER_ROLE");
    uint256 public immutable maturity;
    address public immutable splitter;

    // Claimable yield per YT, accumulated by the splitter
    mapping(address => uint256) public yieldOwed;

    function mint(address to, uint256 amount) external onlyRole(MINTER_ROLE);
    function burn(address from, uint256 amount) external onlyRole(MINTER_ROLE);

    // Claim accrued yield; splitter calls distributeYield() first
    function claimYield() external returns (uint256 amount);
}
```

#### YieldSplitter

The core contract. Holds vault shares, mints PT+YT on deposit, burns them on early exit, redeems at maturity.

```solidity
contract YieldSplitter is AccessControl, ReentrancyGuard {
    IERC4626 public immutable vault;         // e.g. YieldVault
    PrincipalToken public immutable pt;
    YieldToken public immutable yt;
    uint256 public immutable maturity;       // UNIX timestamp

    // Track NAV at each deposit to calculate yield accrued
    uint256 public navAtLastUpdate;          // from NAVOracle
    uint256 public totalVaultShares;         // held by this contract

    // User deposits vault shares → gets PT + YT 1:1 (in asset terms)
    // e.g. 1000 NUSD worth of shares → 1000 PT + 1000 YT
    function split(uint256 vaultShares) external returns (uint256 ptAmount, uint256 ytAmount);

    // Before maturity: burn PT + YT → get vault shares back
    // (costs yield already accrued — you forfeit it)
    function unsplit(uint256 ptAmount) external returns (uint256 vaultShares);

    // After maturity: PT holders redeem 1:1 for underlying
    // YT holders claim all remaining yield
    function redeemPT(uint256 ptAmount) external returns (uint256 assets);
    function redeemYT(uint256 ytAmount) external returns (uint256 yieldAmount);

    // Called by keeper / anyone to distribute accrued yield to YT holders
    function distributeYield() external;

    event Split(address indexed user, uint256 vaultShares, uint256 ptAmount, uint256 ytAmount);
    event Unsplit(address indexed user, uint256 ptAmount, uint256 vaultShares);
    event YieldDistributed(uint256 yieldAmount, uint256 navBefore, uint256 navAfter);
    event PTRedeemed(address indexed user, uint256 ptAmount, uint256 assets);
    event YTRedeemed(address indexed user, uint256 ytAmount, uint256 yield);
}
```

**Yield accounting:**
```
yieldAccrued = (currentNAV - navAtLastUpdate) * totalVaultShares / 1e18
distributeYield() → adds yieldAccrued to ytYieldPool → pro-rata to YT holders
navAtLastUpdate = currentNAV
```

### 5.2 StructuredProduct (Tranches)

**File:** `contracts/derivatives/StructuredProduct.sol`

A pool of vault shares split into two tranches by risk:

```solidity
contract StructuredProduct is AccessControl, ReentrancyGuard {
    IERC4626 public immutable vault;
    IERC20 public immutable seniorToken;    // fixed-yield, first claim
    IERC20 public immutable juniorToken;    // residual yield, loss absorber

    // Target allocation (e.g. 80% senior, 20% junior)
    uint256 public seniorRatioBps;          // e.g. 8000 = 80%

    // Senior gets: base yield up to seniorTargetYieldBps
    // Junior gets: all remaining yield
    uint256 public seniorTargetYieldBps;    // e.g. 300 = 3% APY

    uint256 public maturity;

    function depositSenior(uint256 assets) external returns (uint256 tokens);
    function depositJunior(uint256 assets) external returns (uint256 tokens);
    function redeemSenior(uint256 tokens) external returns (uint256 assets);
    function redeemJunior(uint256 tokens) external returns (uint256 assets);

    // Payout waterfall at maturity:
    // 1. Senior holders get principal + guaranteed yield
    // 2. Junior holders get whatever remains
    function settleAtMaturity() external;

    event SeniorDeposit(address indexed user, uint256 assets, uint256 tokens);
    event JuniorDeposit(address indexed user, uint256 assets, uint256 tokens);
    event MaturitySettled(uint256 seniorPayout, uint256 juniorPayout, uint256 timestamp);
}
```

### 5.3 CreditVault

**File:** `contracts/derivatives/CreditVault.sol`

Borrow NUSD against vault share collateral. Similar to a CDP (collateralized debt position).

```solidity
contract CreditVault is AccessControl, ReentrancyGuard {
    IERC4626 public immutable collateralVault;  // vault shares as collateral
    INexusStableCoin public immutable nusd;      // what we borrow
    INAVOracle public immutable oracle;          // price of collateral

    // Risk parameters (set by ADMIN_ROLE)
    uint256 public collateralRatioBps;           // e.g. 15000 = 150% overcollateralized
    uint256 public liquidationRatioBps;          // e.g. 12000 = 120%
    uint256 public borrowRateBps;                // annual borrow rate, e.g. 500 = 5%

    struct Position {
        uint256 collateralShares;    // vault shares deposited
        uint256 debtNUSD;            // NUSD borrowed
        uint256 lastAccrualTime;     // for interest calculation
    }

    mapping(address => Position) public positions;

    // Deposit vault shares as collateral
    function depositCollateral(uint256 shares) external;

    // Borrow NUSD against collateral (reverts if undercollateralized)
    function borrow(uint256 nusdAmount) external;

    // Repay NUSD debt
    function repay(uint256 nusdAmount) external;

    // Withdraw collateral (only if still above collateralRatio after withdrawal)
    function withdrawCollateral(uint256 shares) external;

    // Anyone can liquidate an undercollateralized position
    // Liquidator repays debt, receives collateral at discount
    function liquidate(address borrower) external;

    // View: current LTV ratio for a position
    function ltvRatio(address borrower) external view returns (uint256 bps);

    event CollateralDeposited(address indexed user, uint256 shares);
    event Borrowed(address indexed user, uint256 nusdAmount);
    event Repaid(address indexed user, uint256 nusdAmount);
    event CollateralWithdrawn(address indexed user, uint256 shares);
    event Liquidated(address indexed borrower, address indexed liquidator, uint256 collateralSeized, uint256 debtRepaid);
}
```

### 5.4 ETFWrapper

**File:** `contracts/derivatives/ETFWrapper.sol`

A basket product holding multiple vaults with fixed allocation weights.

```solidity
contract ETFWrapper is ERC20, AccessControl, ReentrancyGuard {
    struct VaultAllocation {
        address vault;          // IERC4626
        uint256 weightBps;      // e.g. 5000 = 50%
    }

    VaultAllocation[] public allocations;  // must sum to 10000 bps

    // Deposit NUSD → get ETF tokens pro-rata
    // Contract splits NUSD across vaults per weights
    function deposit(uint256 nusdAmount) external returns (uint256 etfTokens);

    // Burn ETF tokens → get NUSD back (withdraws pro-rata from each vault)
    function withdraw(uint256 etfTokens) external returns (uint256 nusdAmount);

    // View: total NAV across all vaults (in NUSD)
    function totalNAV() external view returns (uint256);

    // View: current price per ETF token in NUSD (6 decimals)
    function pricePerToken() external view returns (uint256);

    // Admin: rebalance allocations (requires all withdrawals first, or gradual rebalance)
    function rebalance() external onlyRole(REBALANCER_ROLE);

    event Deposited(address indexed user, uint256 nusdAmount, uint256 etfTokens);
    event Withdrawn(address indexed user, uint256 etfTokens, uint256 nusdAmount);
    event Rebalanced(uint256 timestamp);
}
```
